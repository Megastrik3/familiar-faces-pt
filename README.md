# Problem Statement
Remembering people's names has always been a bit of a challenge for me. Either I don't hear their name properly the first time, or I only see someone once or twice and don't see them on a regular basis. This is especially a difficult challenge for teachers and those who meet lots of people on a regular basis. It can be challenging to recall names quickly when you've already meet 10+ people at that same time. Familiar Faces aims to solve this problem.  

## Project Description
Familiar faces (prototype) is a prototype for a mobile and AR/XR application designed to help users remember names of people that have not seen in a while or that they have just met. It does this by detecting faces of people that you encounter. Then, after a certain number of consistent encounters, or after a given duration, Familiar Faces will ask the user if they want to add that person's face to their list of "Contacts". Once a person is added to the users "Contacts," they will have their name displayed above their head next time the user sees them after it has been a set durration since the last time they have seen them.

## Proposed Method
In order to acheive this goal, several AI models will be used. Since this is an application designed to run on mobile hardware but is being prototyped on a computer, the application will be designed with the hardware constraints of a mobile device in mind, but will forgo these design constraints as necessary to complete the project on time.  

Processing every frame captured by a camera is a very computationally expensive tasks, as such, various "filters" will need to be used to only run detection when a face is actively present. To do this, a lightweight YOLO model will be used to simply detect every face that appears in frame. Using this method, we will be able to identiy faces quility and cheaply, without having to run the face detection algroithm everytime. Then, a Kalman filter will be used to track the movement of the detected face throughout the frame. This will be used to capture only frames where the face is clearly visible to be passed to the face detection algorithm. This filter will also be how the name is displayed above the persons head. Once a person has been positively identified, a textbox will displayed in 2D space above their head. In order to identify a person by their face, the FaceNet algorithm by Google will be used. The algorithm is already optimized for running on mobile devices making it a great choice for this application. The FaceNet model works by encoding detected faces into a high-dimensional vector then saving that vector to a database. Then, a simple distance calculation can be used to see if this face embedding can be used to determine if the face is "familiar" or not.

# Data Sources  
This model uses primarily pretrained model for detecting peoples faces and embedding them into high-dimensional vectors. Additionally, because faces are learned overtime, there is no need to train the FaceNet model on a defined set of faces. However, in order to learn more about model training and to gain more experience in training YOLO models, the YOLO model used to detect faces will be trained localy using [WIDER FACE](http://shuoyang1213.me/WIDERFACE/) dataset. In the event of subpar performance from this custom trained model, the [YOLO-Face](https://github.com/akanametov/yolo-face) model will be used instead.
